Interfaces estáticas y dinámicas

Interfaces estáticas

Una interfaz estática está generalmente basada en HTML. Esto quiere decir que el código se sube a internet para que el usuario pueda interactuar con ella.

Las interfaces estáticas son una buena opción en caso de no actualizar con frecuencia su contenido.

Ventajas

  • Es muy rápida de crear por la sencillez del lenguaje.
  • Tiene una mayor velocidad de carga, ya que se reduce a código HTML y no tiene muchas más funcionalidades.
  • Como el tiempo de respuesta es menor, tiene mejor posicionamiento SEO.

Inconvenientes

  • No tiene muchas opciones de diseño. Sus opciones son muy limitadas.
  • Se necesita tener conocimientos de programación para modificar el HTML.
  • A pesar de poder insertar un ID de Analytics y tener cierta información sobre las interacciones del usuario, no es posible conocer todas las acciones que el usuario realiza

Interfaces dinámicas

Una interfaz dinámica permite cambiar y actualizar su contenido constantemente, ya sea un texto o una imagen, y además interactuar con el usuario de más formas (Comentarios de una publicación, encuestas, ventas de productos…).

Como este tipo de páginas son más complejas, hay CMS (o sistemas de gestión de contenido) que hacen que crearlas sea más sencillo. Por ejemplo: Wordpress.

Las páginas dinámicas suelen combinarse con PHP y MySQL.

Ventajas

  • Si se recurre a un sistema de gestión de contenido para crearla, se puede hacer un gran número de cosas y añadir más opciones mediante plugins.
  • Es fácil de actualizar y con pocos clicks en el CMS es posible cambiar el contenido de la página y su diseño.
  • Tiene muchas más opciones de interacción con el usuario al poder adaptar la información de diferentes formas

Inconvenientes

  • Como tiene más funcionalidades, necesita más recursos que una página estática. También necesita una base de datos en la que se guarda el contenido que se vaya a mostrar.
  • El diseño de la interfez puede estar limitado, ya que el contenido de la misma suele estar basado en información de bases de datos y la presentación de los datos se basa en la estructura de la misma

Ninguna interfaz es necesariamente peor que la otra. Todo depende de para qué se vaya a utilizar
